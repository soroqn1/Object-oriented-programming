# Курсова робота — Планувальник завдань (Python)

## 1. Мета роботи
Показати розуміння ООП, розшарування застосунку (DAL/BLL/PL), інтерфейси та інверсію керування, серіалізацію, валідацію та модульні тести.

## 2. Постановка задачі (Варіант 14)
Керування задачами/виконавцями, призначення, статуси, дедлайни, звіти.

## 3. Архітектура
- **PL (CLI)** — прийом команд, валідація вводу, друк результатів.
- **BLL** — моделі та бізнес-правила (винятки, перевірки, розрахунки).
- **DAL** — сховище JSON через generic-репозиторій.
- **IoC** — `build_container()` зв’язує реалізації.

## 4. ООП
- Класи `Member`, `TaskItem`, `Assignment` (композиція через `Assignment`), поліморфізм/контракти через `IRepository[T]`, власні винятки.
- Узагальнення: `IRepository[T]`.

## 5. Реалізовані кейси
- CRUD для виконавців/задач, призначення з перевіркою завантаження, статуси/прогрес, пошук, підсумковий звіт.

## 6. Валідація і винятки
- BLL: назва задачі, діапазони, дедлайни, перевиділення часу.
- PL: розбір UUID/дат, дружні повідомлення про помилки.

## 7. Тести
pytest (AAA) для ключових сценаріїв `TaskService` та репозиторію.

Див. README.

## 9. Висновки
Архітектура дозволяє безболісно замінити DAL (наприклад на SQLite) без зміни BLL/PL.
